Analytics Workbench Reference Guide

Getting Started

Analytics Workbench is a local-first AI analytics tool. Import any structured dataset, get automatic insights, query with natural language or SQL, and export results with a full audit trail.

Privacy model: AI sees only column names and aggregate statistics. Your row data never leaves your machine. AI generates analysis instructions (SQL) that execute locally via DuckDB.

Four-step workflow:

  1. Import — load a CSV, Excel, TSV, or Parquet file
  2. Insights — AI auto-generates 3-5 analytical observations
  3. Query — ask questions in natural language or write SQL
  4. Export — download results as Excel, TSV, or structured passport

Sidebar layout: The left sidebar has four collapsible sections. Click any section header to expand or collapse it.

  • GET STARTED — Welcome, Reference Guide, Example Cases
  • WORKSPACE — Insights, Query, Saved Queries (auto-expands on first query)
  • DATA — Imported datasets, reference tables, library (auto-expands on first import)
  • SESSIONS — Retrieve/Save sessions and snapshots

Below the sidebar sections: Exit (close immediately) and Save (save session + snapshot, then close).

Importing Data

Click + Import Dataset in the Data section. Supported formats:

  • CSV — comma-separated values (most common)
  • Excel — .xlsx and .xls files
  • TSV — tab-separated values
  • Parquet — columnar format, handles files up to 200M+ rows

All files are converted to local Parquet on import. Nothing is uploaded anywhere. Importing with the same name overwrites the previous version. Large Parquet files use a fast raw-copy path (no decompression).

Insights

When you import a dataset, AI automatically analyzes the column structure and generates 3-5 insight cards. Each card includes:

  • A plain-English headline
  • A brief explanation of why it's notable
  • A supporting chart or summary table
  • An Explore in Query button to load the underlying SQL

Six insight types: Concentration (Pareto), Outliers, Trends, Distribution Skew, Missing/Anomalous Data, Correlation.

Click Refresh Insights to regenerate. Insights are cached — reopening a dataset is instant.

Query

SQL Editor: Write DuckDB SQL directly. Always use dataset as the table name. Do not end queries with a semicolon.

Natural Language: Type a question in the prompt box and click Generate SQL. AI generates DuckDB-compatible SQL from your question.

Suggestions: Click the Suggestions button for AI-generated analytical questions based on your dataset's schema.

Saving Queries: Save frequently-used queries via the Saved Queries panel in Workspace.

DuckDB syntax tips:

  • Use LIKE 'pattern%' instead of = 'value' for string matching (handles trailing special characters in CMS data)
  • Use strftime('%Y-%m', col) for date formatting (format string first)
  • Use col::INTEGER for type casting
  • Use USING SAMPLE 1000 ROWS for sampling large datasets

Reference Tables

Reference tables are small lookup/enrichment tables you JOIN against your primary dataset.

Loading from the Library: Click Library in the Data section to browse pre-built reference tables. Click a table name to load it.

Importing your own: Click + Reference Table to import a CSV, TSV, or Excel file as a reference table. One reference table active at a time.

Using in SQL: Reference tables are available as reference in your queries:

-- Exclude IRA drugs via LEFT JOIN
SELECT d.Brnd_Name, d.Tot_Spndng_2023
FROM dataset d
LEFT JOIN reference r ON d.Brnd_Name = r.drug_name
WHERE r.drug_name IS NULL
ORDER BY d.Tot_Spndng_2023 DESC

String columns in reference tables are automatically title-cased on import so JOINs match without LOWER() wrappers.

Reference Library

Pre-built reference tables shipped with AW:

  • IRA Negotiated Drugs — 35 drugs, Rounds 1-3, with prices and effective dates
  • USP GUARD Categories — 102 drugs mapped to 17 USP therapeutic categories
  • USP GLOBE Categories — 78 drugs mapped to 7 GLOBE payment categories
  • Orphan Drug Status — 65 drugs with FDA orphan designation
  • GLOBE/GUARD/IRA Exclusions — various exclusion lists for policy analysis

To add your own: drop a CSV file into data/reference_library/. It will appear automatically in the Library browser.

Tutorial mode: Step-by-step live replay. Each action (import, reference load, query) executes in real time with narration explaining what's happening and why.

Sessions

The SESSIONS section in the sidebar provides four buttons in two rows:

  • Retrieve Session + Save — browse and restore saved sessions, or save the current session with a name. Session files are saved to data/sessions/ using your chosen name as the filename.
  • Retrieve Snapshot + Save — browse and restore workspace snapshots, or save the current workspace state. Snapshot files are saved to data/snapshots/ using your chosen name.

Sessions vs. Snapshots: A session captures the full event log (every query, import, export). A snapshot captures the current workspace state (which dataset is loaded, which reference table, last SQL query). Use sessions for audit trails. Use snapshots for quick resume.

Auto-snapshot: AW automatically saves workspace state when you exit. On next launch, you'll see a resume prompt offering to restore where you left off.

Exit & Save: The Exit button at the bottom of the sidebar closes AW immediately. The Save button next to it prompts for a name and saves both session and snapshot before closing.

Session Log: AW automatically records every action — imports, queries, reference loads, exports — with timestamps and details. This happens silently in the background.

Exporting

Export Excel / Export TSV: Direct download buttons in the results toolbar. Exports the full query result (up to 200,000 rows).

Result Passport: Click Copy Result Summary to copy a structured JSON profile of your query results to clipboard. Includes per-column statistics, top values, and data quality flags — but zero raw row data. Paste into Claude or any AI assistant for analysis.

Export Passport: Click Export Passport in the Inspect row to download a complete dataset profile JSON. Includes schema, sample values, grain description, quality flags, and starter SQL. Designed for cold-start AI collaboration.

Privacy

Schema-only AI: When AW calls OpenAI, it sends only column names, data types, and aggregate statistics (min, max, mean). Sample rows and raw values are never sent.

What gets sent: Column names, data types, numeric ranges, null rates.

What never leaves your machine: Row data, query results, exported files, Reference Table contents.

Data cleaning: Trailing special characters (asterisks, footnote symbols) are automatically stripped from string values on import, so SQL filters work correctly without manual cleanup.

Troubleshooting

"Table does not exist"Reference table not loaded. Click Library to load it.
"SQL must reference the selected dataset"Use FROM dataset in your query, not the raw filename.
"Parser error"Check DuckDB syntax. Remove trailing semicolons.
Import failsCheck file format. Try saving as CSV if Excel fails.
Insights not generatingCheck that OPENAI_API_KEY is set in your .env file.
